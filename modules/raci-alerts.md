# RACI-алерты

RACI-алерты дают готовый пакет преднастроенных правил оповещения с эскалацией по уровням ответственности. Не нужно вручную заводить десятки правил на каждую метрику: пакет уже покрывает APM, браузерный мониторинг, инфраструктуру и базы данных. Пока нарушение держится, пакет проставляет на оповещении метку уровня эскалации, и по этой метке легко настроить через стандартные политики уведомлений Grafana, кому и когда уходит уведомление.

## Как это работает

Пакет содержит набор правил, каждое из которых покрывает целый класс метрик. Когда правило срабатывает, пакет проставляет на оповещении метку уровня эскалации `gmonit_raci.level`: `L1` сразу, `L2` и `L3` позже, пока нарушение продолжает держаться. Сам пакет уведомления никому не рассылает, он только размечает оповещения уровнями. Маршрутизацию по этим меткам (`L1` дежурной команде, `L2` старшему, `L3` руководству) вы настраиваете через стандартные политики уведомлений Grafana. Так не приходится заводить отдельные правила под каждый уровень: хватает одного условия на метку в политике.

Уровни считаются на стороне базы данных прямо при оценке правила, отдельные сервисы и фоновые процессы не нужны. Параметры конкретного правила (пороги, важность, задержки эскалации) можно переопределить для отдельной сущности, не трогая весь пакет.
