# Настройки — Использование

## Настроить сроки хранения данных

Управление сроками хранения (TTL) позволяет задать, сколько дней хранить каждую категорию данных в системе.

<!-- STEP_GUIDE:START Пошаговое изменение сроков хранения данных: навигация к странице, описание таблицы категорий данных, редактирование значений TTL в днях, сохранение изменений. Скриншоты. -->

1. Нажмите на аватарку в правом верхнем углу и выберите **Настройки**. Если вы находитесь вне интерфейса GMonit (например, на стандартных экранах Grafana), откройте **Administration → Plugins and data → Plugins → GMonit**
2. Перейдите на вкладку **Хранение данных**
3. На странице отображается таблица со всеми категориями данных:

| Категория | Что хранит |
|---|---|
| APM метрики | Агрегированные метрики производительности |
| APM события | Транзакционные события |
| APM ошибки | Данные об ошибках |
| APM SQL трейсы | Медленные SQL-запросы |
| APM логи | Логи приложений |
| APM семплы транзакций | Детальные трейсы транзакций |
| APM custom events | Пользовательские события |
| Трассировки | Распределённые трейсы |
| RUM браузер | События браузерного мониторинга |
| RUM JS ошибки | JavaScript-ошибки в браузере |
| RUM ресурсы | Загрузка ресурсов в браузере |
| Мобильный мониторинг | Данные мобильных приложений |
| Инфраструктура | Метрики и события инфраструктуры |
| OTEL метрики | Метрики OpenTelemetry |

4. В колонке **TTL (дни)** введите нужный срок хранения для каждой категории (минимальное значение — 1 день)
5. Нажмите **Сохранить** — будут обновлены только изменённые категории

Новый TTL применяется только к данным, записанным после изменения. Данные, записанные до изменения, будут удалены по старому сроку.
