# Настройки — Использование

## Настроить сроки хранения данных

Управление сроками хранения (TTL) позволяет задать, сколько дней хранить каждую категорию данных в системе.

<!-- STEP_GUIDE:START Пошаговое изменение сроков хранения данных: навигация к странице, описание таблицы категорий данных, редактирование значений TTL в днях, сохранение изменений. Скриншоты. -->

1. Нажмите на аватарку в правом верхнем углу и выберите **Настройки**. Если вы находитесь вне интерфейса GMONIT (например, на стандартных экранах Grafana), откройте **Administration → Plugins and data → Plugins → GMONIT**
2. Перейдите на вкладку **Хранение данных**
3. На странице отображается таблица со всеми категориями данных:

| Категория | Что хранит |
|---|---|
| APM метрики | Агрегированные метрики производительности |
| APM события | Транзакционные события |
| APM ошибки | Данные об ошибках |
| APM SQL трейсы | Медленные SQL-запросы |
| APM логи | Логи приложений |
| APM семплы транзакций | Детальные трейсы транзакций |
| APM custom events | Пользовательские события |
| Трассировки | Распределённые трейсы |
| RUM браузер | События браузерного мониторинга |
| RUM JS ошибки | JavaScript-ошибки в браузере |
| RUM ресурсы | Загрузка ресурсов в браузере |
| Мобильный мониторинг | Данные мобильных приложений |
| Инфраструктура | Метрики и события инфраструктуры |
| OTEL метрики | Метрики OpenTelemetry |

4. В колонке **TTL (дни)** введите нужный срок хранения для каждой категории (минимальное значение — 1 день)
5. Нажмите **Сохранить** — будут обновлены только изменённые категории

Новый TTL применяется только к данным, записанным после изменения. Данные, записанные до изменения, будут удалены по старому сроку.

## Настроить правила переименования и исключения транзакций

Правила приводят имена транзакций к нужному виду или исключают ненужные — например, схлопывают идентификаторы в путях или убирают служебные транзакции из статистики. Требуется [хранилище настроек](/modules/settings/setup.md#хранилище-настроек-агентов).

<!-- STEP_GUIDE:START Управление правилами обработки транзакций в интерфейсе настроек: выбор области применения (приложение или глобально), создание правила с регулярным выражением и действием (замена или исключение), порядок применения и прерывание цепочки, активация, изменение и удаление правил. Скриншоты. -->

1. Нажмите на аватарку в правом верхнем углу и выберите **Настройки**. Если вы вне интерфейса GMONIT, откройте **Administration → Plugins and data → Plugins → GMONIT**.
2. Перейдите на вкладку **Правила транзакций** — на странице таблица существующих правил (область, регулярное выражение, действие, порядок) и кнопка **Создать правило**.
3. Нажмите **Создать правило** и заполните форму:
   - **Область** — приложение, к которому применяется правило, или **Все приложения (global)**.
   - **Регулярное выражение** — шаблон имени транзакции, например `^WebTransaction/(.*)`.
   - **Действие**:
     - **Заменить** — переименовать совпавшие транзакции по шаблону из поля **Замена** (например `WebTransaction/Custom/\1`, где `\1` — группа из регулярного выражения).
     - **Игнорировать** — исключить совпавшие транзакции из статистики.
   - **Порядок (eval_order)** — очередность применения: чем меньше значение, тем раньше применяется правило.
   - **Прервать цепочку** — если включено, после срабатывания этого правила следующие не применяются.
   - **Активно** — включает или выключает правило без удаления.
   - **Комментарий** — произвольное описание.
4. Сохраните правило — оно появится в таблице. Правила применяются в порядке возрастания **eval_order**.

Чтобы изменить правило, нажмите на его строку; чтобы удалить — используйте действие удаления и подтвердите.
