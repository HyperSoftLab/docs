# Как включить настройки

## Включение управления хранением данных

Управление сроками хранения через интерфейс по умолчанию отключено и требует включения через переменную окружения коллектора. В lite-версии GMonit управление недоступно — используются фиксированные сроки хранения.

<!-- STEP_GUIDE:START Параметры включения управления хранением данных: переменная окружения для активации, поведение при отключённом флаге, ограничения lite-версии, значения TTL по умолчанию. -->

Для включения управления хранением данных задайте переменную окружения коллектора:

```
TTL_UI_ENABLED=true
```

По умолчанию значение — `false`. Когда управление отключено, на странице настроек отображается сообщение: «Управление TTL отключено. Установите TTL_UI_ENABLED=true в коллекторе для управления сроками хранения данных.»

### Ограничения lite-версии

В lite-версии GMonit управление сроками хранения недоступно независимо от значения `TTL_UI_ENABLED`. Для всех категорий данных используется фиксированный TTL — **3 дня**.

### Значения TTL по умолчанию

Если TTL не изменён через интерфейс, используются значения, заданные при развёртывании:

| Категория данных | TTL по умолчанию |
|---|---|
| APM метрики | 30 дней |
| APM события | 30 дней |
| APM ошибки | 30 дней |
| APM SQL трейсы | 30 дней |
| APM логи | 1 день |
| APM семплы транзакций | 8 дней |
| APM custom events | 30 дней |
| Трассировки | 8 дней |
| RUM браузер | 8 дней |
| RUM JS ошибки | 8 дней |
| RUM ресурсы | 2 дня |
| Мобильный мониторинг | 1 день |
| Инфраструктура | 30 дней |
| OTEL метрики | 30 дней |
