# Коробочные алерты — Настройка

## Включение пакета алертов и интервал оценки

По умолчанию коробочный пакет алертов **стоит на паузе**: после установки ни одно правило не срабатывает, пока пакет не включат явно. Интервал оценки правил по умолчанию равен **1 минуте**.

Поведение пакета задают две переменные окружения Grafana:

| Переменная | Описание | По умолчанию |
|---|---|---|
| `GMONIT_GRAFANA_DEFAULT_ALERT_PACK_ENABLED` | Включает пакет. При значении `true` все правила пакета снимаются с паузы. Любое другое значение или отсутствие переменной оставляет пакет на паузе | `false` (на паузе) |
| `GMONIT_GRAFANA_DEFAULT_ALERT_PACK_INTERVAL` | Переопределяет интервал оценки всех правил пакета | `1m` |

### Как правильно задать интервал

Значение `GMONIT_GRAFANA_DEFAULT_ALERT_PACK_INTERVAL` подставляется в правила как есть, поэтому указывайте его в формате длительности Grafana (например `30s`, `1m`, `2m`).

Требования к значению:

- **Кратно базовому интервалу планировщика (10 секунд).** Значения вроде `25s` или `1m5s` некорректны. Допустимы `10s`, `30s`, `1m`, `90s`, `2m` и так далее.
- **Не равно нулю.** Пустое или нулевое значение тоже некорректно.

Если значение указано некорректно, Grafana **не запустится**: провижининг правил пакета упадёт на старте. Поэтому после изменения интервала проверьте, что контейнер Grafana поднялся.

Чем меньше интервал, тем быстрее обнаруживаются нарушения и точнее сроки эскалации, но выше нагрузка на оценку правил. Для большинства установок значение по умолчанию `1m` оптимально; уменьшать его стоит только ради более быстрой реакции и при запасе по ресурсам.

Пример включения пакета с оценкой раз в 30 секунд:

```yaml
environment:
  GMONIT_GRAFANA_DEFAULT_ALERT_PACK_ENABLED: "true"
  GMONIT_GRAFANA_DEFAULT_ALERT_PACK_INTERVAL: 30s
```

## Маршрутизация уведомлений по уровням эскалации RACI

Эскалация по матрице RACI работает только для правил из коробочного пакета. Каждому уровню эскалации соответствует метка на оповещении, и доставку оповещений нужным командам вы настраиваете через стандартные политики уведомлений Grafana с маршрутизацией по этим меткам.

<!-- STEP_GUIDE:START Маршрутизация оповещений по уровням эскалации через политики уведомлений Grafana: какие метки несут уровень эскалации, как настроить дерево политик и точки контакта для разных уровней. -->

Каждое сработавшее оповещение пакета несёт две метки:

| Метка | Значения | Что означает |
|---|---|---|
| `gmonit_raci.level` | `L1`, `L2`, `L3` | Текущий уровень эскалации. `L1` присваивается сразу, `L2` и `L3` позже, пока нарушение держится |
| `gmonit_raci.severity` | `P1`, `P2`, `P3` | Профиль важности оповещения, задаёт скорость эскалации |

Одно нарушение порождает отдельное оповещение на каждый активный уровень. Так разные уровни можно направить разным получателям: `L1` дежурной команде, `L2` старшему дежурному, `L3` руководству.

Маршрутизация настраивается через стандартные политики уведомлений Grafana:

1. Создайте точки контакта (contact points) для каждого получателя, например отдельные каналы для дежурной команды, старшего дежурного и руководства.

2. Откройте раздел оповещений Grafana → **Notification policies**.

3. Добавьте вложенную политику с условием по метке `gmonit_raci.level`:
   - `gmonit_raci.level = L1` → точка контакта дежурной команды
   - `gmonit_raci.level = L2` → точка контакта старшего дежурного
   - `gmonit_raci.level = L3` → точка контакта руководства

4. При необходимости уточните маршрут по метке `gmonit_raci.severity`, например чтобы критические оповещения (`P1`) шли по более срочному каналу.

> Сейчас маршрутизация настраивается вручную через интерфейс Grafana. Настройка прямо в интерфейсе GMONIT пока в планах.
