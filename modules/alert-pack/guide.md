# Коробочные оповещения — Использование

## Состав коробочного пакета

Готовые правила, которые включаются вместе с пакетом. Пороги ниже — значения по умолчанию; любое из них можно [переопределить для отдельной сущности](#Переопределить-параметры-оповещения-для-сущности).

Пакет покрывает четыре области мониторинга. Часть правил объединяет несколько связанных метрик под одним оповещением — тогда у каждой метрики свой порог по умолчанию, и переопределять их можно по отдельности.

Если не указано иное, правило срабатывает при **превышении** порога.

### Потеря связи с агентом

Эти правила следят за самим фактом поступления данных и срабатывают, когда агент перестаёт присылать метрики. Порога как такового у них нет — реагируют на отсутствие данных.

| Оповещение | Когда срабатывает |
|---|---|
| APM: нет данных от агента | Приложение перестало присылать APM-метрики |
| Database: нет данных от агента | Агент базы данных перестал присылать метрики |
| Host: нет данных от агента | Хост перестал присылать метрики |
| Container: нет данных от агента | Контейнер перестал присылать метрики |
| Storage: нет данных от агента | Агент хранилища перестал присылать метрики |

## Просмотреть активные оповещения

<!-- STEP_GUIDE:START Просмотр сработавших оповещений пакета: где находится список, как отображается текущий уровень эскалации нарушения, какие метки и сведения сопровождают каждое оповещение. -->

Правила пакета — это обычные оповещения Grafana, поэтому их состояние видно в стандартном разделе оповещений Grafana. Каждое сработавшее правило показывает свои активные инстансы (серии).

Сработавшие оповещения по умолчанию попадают и на общий дашборд GMONIT, в список активных оповещений, так что их видно сразу при входе, без перехода в раздел оповещений Grafana.

### Уровни эскалации RACI

Дополнительно к самому факту срабатывания пакет проставляет на каждом инстансе метки эскалации по матрице RACI:

| Метка | Значения | Что означает |
|---|---|---|
| `gmonit_raci.level` | `L1`, `L2`, `L3` | Текущий уровень эскалации нарушения |
| `gmonit_raci.severity` | `P1`, `P2`, `P3` | Профиль важности, задаёт скорость эскалации |

Как читать уровни:

- **`L1`** присваивается сразу при срабатывании.
- **`L2`** появляется, когда нарушение держится близко к заданной для уровня задержке.
- **`L3`** появляется при дальнейшем сохранении нарушения.

Одно и то же нарушение отображается несколькими инстансами, по одному на каждый активный уровень. Поэтому при затяжной проблеме у правила одновременно видны инстансы с `L1`, `L2` и `L3` для одной и той же серии. Так задумано: при смене уровня предыдущие инстансы не помечаются как «разрешённые» ложно.

## Переопределить параметры оповещения для сущности

> Если включён [режим ограничения по командам](/modules/alert-pack/setup.md#ограничение-переопределений-по-командам), в списке и в форме доступны только переопределения вашей команды Grafana.

<!-- STEP_GUIDE:START Переопределение параметров срабатывания правила для отдельной сущности: где находится страница, какие параметры доступны (порог, исключение из оповещений, условие применения), как создать, изменить и удалить переопределение. -->

Переопределения меняют параметры срабатывания правила для отдельной сущности (приложения, хоста, метрики), не трогая весь пакет. Они собраны на странице **Оповещения → Переопределения**.

На странице показана таблица существующих правил со столбцами: **Область применения** (условие), **Оповещение**, **Состояние**, **Порог**. Сверху доступны фильтры по оповещению и состоянию.

### Создать переопределение

1. Нажмите **Создать правило** в правом верхнем углу, откроется форма.

2. **Оповещение**: выберите правило, к которому применяется переопределение. Вариант **Глобальное правило** применяет его ко всем правилам пакета.

3. **Область применения**: задайте условия на метки, чтобы ограничить переопределение конкретными сущностями. Нажмите **Добавить условие**, выберите метку, оператор и значения. Без условий переопределение применяется ко всем сериям правила.

4. **Переопределение**: задайте, что именно заменить.
   - **Исключить из оповещений**: переключатель, попавшие под область серии перестают порождать оповещения.
   - **Порог оповещения**: числовое значение порога срабатывания. Под полем показан порог по умолчанию для выбранной метрики.

5. **Описание**: произвольный комментарий о назначении правила.

6. Нажмите **Создать**. Правило появится в таблице.

### Изменить или удалить переопределение

1. В таблице нажмите на строку нужного правила (столбец **Область применения**), откроется та же форма с заполненными значениями.
2. Измените параметры и нажмите **Сохранить**, либо нажмите **Удалить** для удаления правила (потребуется подтверждение).

## Настроить эскалацию оповещения

> Если включён [режим ограничения по командам](/modules/alert-pack/setup.md#ограничение-переопределений-по-командам), в списке и в форме доступны только правила вашей команды Grafana.

Эскалация задаёт, как быстро нарушение поднимается по уровням `L1` → `L2` → `L3`. Профиль важности и задержки уровней настраиваются для отдельной сущности, не трогая весь пакет.

<!-- STEP_GUIDE:START Настройка профиля важности и задержек уровней эскалации для отдельной сущности: где находится страница, какие параметры доступны (профиль важности, задержки уровней, условие применения), значения по умолчанию по профилям, как создать и изменить правило. -->

Правила эскалации собраны на странице **Оповещения → Эскалация**. Таблица показывает столбцы: **Область применения**, **Оповещение**, **Профиль**, **Критичность**, **L1**, **L2**, **L3**. Сверху доступны фильтры по оповещению и профилю.

### Создать правило эскалации

1. Нажмите **Создать правило** в правом верхнем углу, откроется форма.

2. **Оповещение**: выберите правило, к которому применяется настройка. Вариант **Глобальное правило** применяет её ко всем правилам пакета.

3. **Область применения**: задайте условия на метки, чтобы ограничить правило конкретными сущностями. Без условий оно применяется ко всем сериям правила.

4. **Переопределение**: задайте параметры эскалации.
   - **Профиль эскалации**: `P1 — критический`, `P2 — предупреждение` или `P3 — информация`. Профиль задаёт скорость эскалации.
   - **Пороги эскалации**: поля **L2** и **L3** — сколько минут нарушение должно непрерывно длиться, прежде чем поднимется уровень (целое число, 0–120). **L1** всегда `0` (присваивается сразу). Пустое поле означает «использовать значение профиля по умолчанию».

   Значения по умолчанию по профилям (в минутах):

   | Профиль | L1 | L2 | L3 |
   |---|---|---|---|
   | P1 | 0 | 15 | 30 |
   | P2 | 0 | 30 | 60 |
   | P3 | 0 | 60 | 120 |

5. **Описание**: произвольный комментарий о назначении правила.

6. Нажмите **Создать**. Правило появится в таблице. Изменение и удаление — по клику на строку, как и для переопределений.
