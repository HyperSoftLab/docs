# Коробочные оповещения — Использование

## Просмотреть активные оповещения

<!-- STEP_GUIDE:START Просмотр сработавших оповещений пакета: где находится список, как отображается текущий уровень эскалации нарушения, какие метки и сведения сопровождают каждое оповещение. -->

Правила пакета — это обычные оповещения Grafana, поэтому их состояние видно в стандартном разделе оповещений Grafana. Каждое сработавшее правило показывает свои активные инстансы (серии).

Сработавшие оповещения по умолчанию попадают и на общий дашборд GMONIT, в список активных оповещений, так что их видно сразу при входе, без перехода в раздел оповещений Grafana.

### Уровни эскалации RACI

Дополнительно к самому факту срабатывания пакет проставляет на каждом инстансе метки эскалации по матрице RACI:

| Метка | Значения | Что означает |
|---|---|---|
| `gmonit_raci.level` | `L1`, `L2`, `L3` | Текущий уровень эскалации нарушения |
| `gmonit_raci.severity` | `P1`, `P2`, `P3` | Профиль важности, задаёт скорость эскалации |

Как читать уровни:

- **`L1`** присваивается сразу при срабатывании.
- **`L2`** появляется, когда нарушение держится близко к заданной для уровня задержке.
- **`L3`** появляется при дальнейшем сохранении нарушения.

Одно и то же нарушение отображается несколькими инстансами, по одному на каждый активный уровень. Поэтому при затяжной проблеме у правила одновременно видны инстансы с `L1`, `L2` и `L3` для одной и той же серии. Так задумано: при смене уровня предыдущие инстансы не помечаются как «разрешённые» ложно.

## Переопределить параметры оповещения для сущности

<!-- STEP_GUIDE:START Переопределение параметров правила для отдельной сущности: как открыть форму переопределения, какие параметры доступны (пороги, важность, задержки уровней эскалации, условие применения, включение и выключение), как сохранить переопределение, как роль Grafana влияет на доступ (Viewer — только просмотр, всегда) и как режим разграничения доступа по командам меняет форму (заблокированная строка команды в области применения). -->

Переопределения меняют параметры правила для отдельной сущности (приложения, хоста, метрики), не трогая весь пакет. Они собраны на странице **Переопределения оповещений**.

На странице показана таблица существующих правил со столбцами: **Область** (условие применения), **Оповещение**, **Состояние**, **Порог**, **Профиль**, **L2**, **L3**. Сверху доступны фильтры по оповещению, профилю эскалации и состоянию.

### Создать переопределение

1. Нажмите **Добавить правило** в правом верхнем углу, откроется форма из четырёх секций.

2. **Оповещение**: выберите правило, к которому применяется переопределение. Вариант **Глобальное правило** применяет его ко всем правилам пакета.

3. **Область применения**: задайте условия на метки, чтобы ограничить переопределение конкретными сущностями. Нажмите **Добавить условие**, выберите метку, оператор и значения. Без условий переопределение применяется ко всем сериям правила.

4. **Переопределение**: задайте, что именно заменить.
   - **Исключить из оповещений**: переключатель, попавшие под область серии перестают порождать оповещения.
   - **Порог оповещения**: числовое значение порога срабатывания. Под полем показан порог по умолчанию для выбранной метрики.
   - **Профиль эскалации**: `P1`, `P2` или `P3`. Профиль задаёт скорость эскалации.
   - **Пороги эскалации**: поля **L2** и **L3**, сколько минут нарушение должно непрерывно длиться, прежде чем поднимется уровень (целое число, 0–120). **L1** всегда `0` (присваивается сразу). Пустое поле означает «использовать значение профиля по умолчанию».

   Значения по умолчанию по профилям (в минутах):

   | Профиль | L1 | L2 | L3 |
   |---|---|---|---|
   | P1 | 0 | 15 | 30 |
   | P2 | 0 | 30 | 60 |
   | P3 | 0 | 60 | 120 |

5. **Описание**: произвольный комментарий о назначении правила.

6. Нажмите **Создать**. Правило появится в таблице.

### Изменить или удалить переопределение

1. В таблице нажмите на строку нужного правила (столбец **Область**), откроется та же форма с заполненными значениями.
2. Измените параметры и нажмите **Сохранить**, либо нажмите **Удалить** для удаления правила (потребуется подтверждение).

### Доступ по ролям и командам

Страница и форма всегда учитывают вашу роль Grafana:

- **Viewer** может только просматривать правила: кнопка **Добавить правило** скрыта, в открытой форме нет кнопок сохранения и удаления, вместо **Отмена** — **Закрыть**.
- **Editor** и **Admin** могут создавать, изменять и удалять правила.

Дополнительно администратор может включить режим разграничения доступа к переопределениям по командам Grafana. Режим включается переменной окружения `RESTRICT_ACCESS_TO_TEAMS` коллектора (задаётся в его конфигурации, по умолчанию выключен).

Разграничение работает через лейбл `team`: у сущности должен быть лейбл `team` со значением, равным названию команды в Grafana. По этому совпадению система понимает, к какой команде относится сущность, и показывает её правила соответствующим участникам. Если лейбла `team` нет или его значение не совпадает ни с одной командой, правила для такой сущности не увидит ни один участник команды — только Admin без назначенных команд, которому доступны все правила.

При включённом режиме страница и форма дополнительно учитывают ваши команды:

- **Editor** видит в таблице только правила, чья область применения затрагивает его команды. В форме, в секции **Область применения**, строка с командой заблокирована: значение ограничено вашими командами, удалить строку нельзя — так правило нельзя распространить на чужие команды.
- **Admin** без назначенных команд работает со всеми правилами без ограничений; заблокированной строки команды у него нет.

Чтобы коллектор мог корректно резолвить команды пользователей, ему нужен доступ к Grafana API через сервисный аккаунт с ролью **Admin** — чтение команд и их участников требует админских привилегий. Прокиньте в коллектор переменные окружения `GRAFANA_API_URL` и `GRAFANA_ADMIN_API_AUTHORIZATION`.

Если пользователь не состоит ни в одной команде (и не Admin), сохранять правила нельзя. При попытке сохранить правило вне зоны доступа сохранение отклоняется с пояснением, например «Правило затрагивает команды, в которых вы не состоите» или «Правило принадлежит другой команде».
