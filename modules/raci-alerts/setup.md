# RACI-алерты — Настройка

## Включение пакета алертов и интервал оценки

По умолчанию пакет RACI-алертов **поставлен на паузу** — после установки ни одно правило не срабатывает, пока пакет не включён явно. Интервал оценки правил по умолчанию равен **1 минуте**.

Поведение пакета задаётся двумя переменными окружения Grafana:

| Переменная | Описание | По умолчанию |
|---|---|---|
| `GMONIT_GRAFANA_DEFAULT_ALERT_PACK_ENABLED` | Включает пакет. При значении `true` все правила пакета снимаются с паузы. Любое другое значение или отсутствие переменной оставляет пакет на паузе | `false` (на паузе) |
| `GMONIT_GRAFANA_DEFAULT_ALERT_PACK_INTERVAL` | Переопределяет интервал оценки всех правил пакета | `1m` |

### Как правильно задать интервал

Значение `GMONIT_GRAFANA_DEFAULT_ALERT_PACK_INTERVAL` подставляется в правила как есть, поэтому его нужно указывать в формате длительности Grafana (например `30s`, `1m`, `2m`).

Требования к значению:

- **Кратно базовому интервалу планировщика (10 секунд).** Значения вроде `25s` или `1m5s` некорректны и приведут к ошибке оценки правил. Допустимы `10s`, `30s`, `1m`, `90s`, `2m` и т.д.
- **Не равно нулю.** Пустое или нулевое значение оставит правила без оценки.

Меньший интервал означает более быстрое обнаружение нарушений и более точные сроки эскалации, но повышает нагрузку на оценку правил. Для большинства установок значение по умолчанию `1m` оптимально; уменьшать его стоит только при необходимости более быстрой реакции и при наличии запаса по ресурсам.

Пример включения пакета с оценкой раз в 30 секунд:

```yaml
environment:
  GMONIT_GRAFANA_DEFAULT_ALERT_PACK_ENABLED: "true"
  GMONIT_GRAFANA_DEFAULT_ALERT_PACK_INTERVAL: 30s
```

## Маршрутизация уведомлений по уровням эскалации

Каждому уровню эскалации соответствует метка на оповещении. Доставка оповещений нужным командам настраивается через стандартные политики уведомлений Grafana с маршрутизацией по этим меткам.

<!-- STEP_GUIDE:START Маршрутизация оповещений по уровням эскалации через политики уведомлений Grafana: какие метки несут уровень эскалации, как настроить дерево политик и точки контакта для разных уровней. -->

Каждое сработавшее оповещение пакета несёт две метки:

| Метка | Значения | Что означает |
|---|---|---|
| `gmonit_raci.level` | `L1`, `L2`, `L3` | Текущий уровень эскалации. `L1` присваивается сразу, `L2` и `L3` — по мере того, как нарушение сохраняется во времени |
| `gmonit_raci.severity` | `P1`, `P2`, `P3` | Профиль важности оповещения, задающий скорость эскалации |

Одно нарушение порождает отдельное оповещение на каждый активный уровень. Это позволяет направить разные уровни разным получателям: `L1` — дежурной команде, `L2` — старшему дежурному, `L3` — руководству.

Маршрутизация настраивается через стандартные политики уведомлений Grafana:

1. Создайте точки контакта (contact points) для каждого получателя — например, отдельные каналы для дежурной команды, старшего дежурного и руководства.

2. Откройте раздел оповещений Grafana → **Notification policies**.

3. Добавьте вложенную политику с условием по метке `gmonit_raci.level`:
   - `gmonit_raci.level = L1` → точка контакта дежурной команды
   - `gmonit_raci.level = L2` → точка контакта старшего дежурного
   - `gmonit_raci.level = L3` → точка контакта руководства

4. При необходимости уточните маршрут по метке `gmonit_raci.severity` — например, чтобы критические оповещения (`P1`) шли по более срочному каналу.

> Маршрутизация настраивается вручную через интерфейс Grafana. Автоматическая настройка прямо в интерфейсе GMonit — в планах.
